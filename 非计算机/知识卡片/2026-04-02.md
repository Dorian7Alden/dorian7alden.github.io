【后端规范】【Spring】XML 配置加载的核心三步流程

---
Spring 加载配置的本质是将**外部化配置 (XML/Properties)** 转化为容器内部的 **`BeanDefinition`** 对象。

1.  **资源定位 (Resource Location)**: `ResourceLoader` 将配置文件路径 (如 `classpath:app.xml`) 转换为 `Resource` 对象。
2.  **资源加载 (Resource Loading)**: `BeanDefinitionReader` (如 `XmlBeanDefinitionReader`) 读取 `Resource` 的内容。
3.  **解析与注册 (Parsing & Registration)**: `BeanDefinitionParser` 将 XML 标签解析成 `BeanDefinition`，并通过 `BeanDefinitionRegistry` 注册到 `BeanFactory` 中。



【后端规范】【Spring Boot】自动配置核心加载机制

---
Spring Boot 的自动配置简化了传统 XML 配置，其核心入口是 `@SpringBootApplication`。

1.  **加载入口**: 通过 `SpringFactoriesLoader` 扫描所有 `jar` 包下的 `META-INF/spring.factories` 文件，获取所有自动配置类。
2.  **条件装配**: 根据 `@ConditionalOn...` 注解判断是否满足条件，决定是否加载该自动配置类。
3.  **属性绑定**: 通过 `@ConfigurationProperties` 将 `application.yml` 或 `application.properties` 中的属性值绑定到对应的 Java Bean 上。



【后端规范】【Spring Boot】YAML 配置文件书写规范

---
YAML 因其简洁的语法成为 Spring Boot 配置的主流选择，但其格式要求严格。

- **缩进敏感**: 必须使用 **2 个空格**进行缩进，严禁使用 `Tab` 键，否则会导致解析失败。
- **键名命名**: 推荐使用**烤肉串式 (kebab-case)**，例如 `server.port` 或 `spring.datasource.url`。
- **数据类型**:
    - 字符串**无需**加引号，除非内容包含空格或特殊字符。
    - 使用 `:` 结尾，后跟一个空格，再写值。



【最佳实践】【Spring】配置项注入的最佳实践

---
- **优先使用 `@ConfigurationProperties`**: 当需要注入多个相关属性时，创建一个配置类并使用 `@ConfigurationProperties` 进行批量注入。这比多个 `@Value` 更清晰、类型安全。
- **使用 `@Value` 注入单个值**: 对于单个、独立的配置项，可以使用 `@Value("${property.key}")`。
- **必须设置默认值**: 为防止因配置缺失导致应用启动失败，应始终提供默认值。
    - 示例: `@Value("${app.timeout:5000}")`，当 `app.timeout` 未配置时，默认值为 `5000`。



【企业级实践】【微服务】配置中心的核心价值

---
在微服务架构中，使用 **Spring Cloud Config**、**Nacos** 或 **Apollo** 等配置中心是最佳实践。

- **统一管理**: 所有微服务的配置都集中存储在配置中心 (如 Git 仓库或 Nacos 服务)，便于维护和审计。
- **动态刷新**: 修改配置后，服务无需重启即可生效。这通常需要配合 `@RefreshScope` 注解使用，它会在配置变更时销毁并重建 Bean。
- **环境隔离**: 配置中心天然支持多环境 (dev, test, prod) 的配置隔离。



【企业级实践】【安全】配置文件中敏感信息的加密方案

---
数据库密码、API Key 等敏感信息严禁明文存储在配置文件中。

- **推荐方案**: 使用 **Jasypt (Java Simplified Encryption)** 库。
- **工作原理**:
    1.  在 `pom.xml` 中引入 `jasypt-spring-boot-starter` 依赖。
    2.  在配置文件中设置加密密钥 (salt)。
    3.  将敏感信息加密后，以 `ENC(加密后的密文)` 的格式写入配置文件。
- **自动解密**: Spring Boot 应用启动时，Jasypt 会自动拦截并解密 `ENC(...)` 包裹的值，业务代码无感知。



【企业级实践】【Spring Boot】多环境配置 (Profile) 管理策略

---

Profile 是 Spring 提供用于隔离不同环境配置的核心功能。

- **文件命名**: `application-{profile}.yml` (例如 `application-dev.yml`, `application-prod.yml`)。`application.yml` 中存放通用配置。
- **激活方式**:
    1.  **配置文件**: 在 `application.yml` 中设置 `spring.profiles.active=dev`。
    2.  **启动参数 (推荐)**: `java -jar app.jar --spring.profiles.active=prod`。这种方式优先级更高，无需修改代码即可切换环境。
- **严格隔离**: 确保 `dev`、`test`、`uat`、`prod` 等环境的配置完全独立，特别是数据库、中间件等连接信息。
