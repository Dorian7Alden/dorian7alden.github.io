【职场/管理】**能量与能力**的认知差异

---

在职业发展中，**能量**往往比**能力**更具决定性。

- **能量**: 指一个人的内在动力、抗压能力、积极心态以及对他人的感染力。
- **能力**: 指执行具体任务的技术、知识和熟练度。
> 💡 **感悟**: 能力决定了你能走多快，而能量决定了你能走多远。



【商业模式】**顾客、用户、客户与影响者**的区别

---

在产品逻辑和商业环境中，需精准区分不同角色的定义：

- **用户**: 产品的最终使用者（User）。
- **顾客**: 支付金钱购买产品的人（Customer）。
- **客户**: 长期合作并产生商业价值的对象。
- **影响者**: 不直接参与买卖或使用，但能左右购买决策的人。



【设计模式】**工厂模式**与**创建者模式**的异同

---

两者都属于创建型设计模式，但关注点不同。

- **相同点**: 都是为了隐藏对象创建的复杂逻辑，将对象的创建与使用解耦。
- **不同点**:
  - **工厂模式**: 关注的是“**选哪一个**”，即根据条件返回不同子类的实例。
  - **创建者模式**: 关注的是“**如何构建**”，即将一个复杂对象的构建过程与其表示分离。



【后端框架】Spring IoC 容器：**基于 XML 获取 Bean** 的方式

---

在 XML 配置文件中，通过 `<bean>` 标签定义对象，Spring 提供了多种获取方式。

- **基础配置属性**:
  - `id`: Bean 的唯一标识。
  - `class`: Bean 对应的全类名。
- **FactoryBean 机制**:
  - 实现 `FactoryBean<?>` 接口。
  - 作用：整合第三方框架的常用机制，减少繁琐的 XML 配置操作。



【后端框架】Spring IoC 容器：**Bean 的依赖注入（DI）**方式

---

DI（Dependency Injection）是 IoC 的具体实现，主要通过以下方式为属性赋值：

- **初始化注入**:
  - **Setter 注入**: 使用 `<property>` 标签。
  - **构造器注入**: 使用 `<constructor-arg>` 标签。
- **属性赋值细节**:
  - **普通引用**: 使用 `ref` 属性。
  - **级联属性**: 如 `clazz.clazzname` 形式。
  - **p 命名空间**: 通过 `<bean p:property="xxx">` 简化配置。
  - **外部配置**: 利用 `<context>` 空间注入 `.properties` 文件中的 `${a.b}`。



【后端框架】Spring IoC 容器：**集合类型属性**的赋值

---

Spring 支持对复杂集合类型的注入：

- **数组类**: 使用 `<array>` 和 `<value>`。
- **List 类**: 使用 `<list>` 和 `<ref>`。
- **Map 类**: 使用 `<entry>` 配合 `<key>`、`<value>` 或 `<ref>`。
- **引用集合**: 使用 `<util>` 名称空间定义独立的集合 Bean 并进行引用。



【后端框架】Spring IoC 容器：**特殊符号与空值**处理

---

在 XML 中处理特殊字符时需要转义或使用特定标签：

- **空值**: 使用 `<null/>` 标签。
- **转义字符**: 例如使用 `&lt;` 表示 `<`。
- **CDATA 区块**: 使用 `<![CDATA[ a < b ]]>` 原样输出内容。



【后端框架】Spring IoC 容器：**Bean 的作用域（Scope）**

---

作用域决定了 Bean 实例的生命周期和创建时机：

- **singleton（单例）**: 默认值。IoC 容器**初始化时**创建，全局唯一。
- **prototype（多例）**: 每次**获取 Bean 时**创建新实例。

【后端框架】Spring IoC 容器：**Bean 的完整生命周期**

---

Bean 从创建到销毁经历以下 7 个核心阶段：

1. **实例化**: 调用无参构造器创建对象。
2. **属性赋值**: 为 Bean 设置属性（DI）。
3. **后置处理（前）**: `BeanPostProcessor` 在初始化前执行。
4. **初始化**: 执行指定的 `init-method`。
5. **后置处理（后）**: `BeanPostProcessor` 在初始化后执行。
6. **就绪**: Bean 准备就绪，可被程序使用。
7. **销毁**: 容器关闭时执行指定的 `destroy-method`。



【算法经验】**快速排序（Quick Sort）**的核心原理

---

快速排序是一种高效的排序算法，基于**分治策略**实现。

- **核心思想**: 通过一趟排序将待排记录分隔成独立的两部分，其中一部分记录的关键字均比另一部分的关键字小，则可分别对这两部分记录继续进行排序，以达到整个序列有序。
- **关键步骤**:
  1. **基准数选择**: 选择一个数作为基准（Pivot）。
  2. **哨兵划分**: 将大于基准的移到右边，小于基准的移到左边。
  3. **递归排序**: 对左右子区间重复上述过程。



【算法经验】快速排序的**性能优势与复杂度**分析

---

- **时间复杂度**:
  - 平均/最好：$O(n \log n)$。
  - 最坏：$O(n^2)$（当基准数选得极差时）。
- **优势**:
  - **缓存友好**: 执行划分操作时，子数组连续加载到缓存，访问效率高。
  - **常数系数小**: 比较和交换的总操作量相对较少。
  - **概率论**: 出现最差情况的概率极低。



【算法经验】快速排序：**哨兵划分（Partition）**的逻辑细节

---

划分过程通常使用双指针（i, j）进行。

- **指针相遇逻辑**:
  - 若以 `left` 为基准，应**先动右指针 j**。
  - `j` 寻找比基准小的数，`i` 寻找比基准大的数。
  - 当 `i == j` 时，交换基准数与当前指针位置的数。
- **原因**: 先动 `j` 保证了相遇点处的数值一定**小于或等于**基准数，从而确保交换后基准数左侧全部小于基准。

```cpp
int partition(vector<int> &nums, int left, int right) {
    int i = left, j = right;
    while (i < j) {
        while (i < j && nums[j] >= nums[left]) --j;
        while (i < j && nums[i] <= nums[left]) ++i;
        swap(nums[i], nums[j]);
    }
    swap(nums[i], nums[left]);
    return i;
}
```
