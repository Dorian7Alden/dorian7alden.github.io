## 目录标签化管理规范



> 【强制】：必须遵守的内容
>
> 【推荐】：最好遵守的内容，特殊条件下可以自行调整



### 核心原则

1. 【强制】**目录即标签**：每个顶层目录代表一个分类标签。一个文档只属于一个顶层目录，通过目录名实现分类。

   说明：目录名 = 文档的唯一定位标签。`学习/Java学习/Spring.md` 的标签是"学习"，不是"Java学习"。子目录仅用于标签内部的细分组。

2. 【强制】**根目录干净原则**：仓库根目录下只保留以下文件，其余文档必须放入对应的顶层目录：

   - `README.md` — 仓库说明
   - `CLAUDE.md` — Claude Code 配置
   - `.gitignore` — Git 忽略规则
   - `.docs/` — 文档规范（本目录）
   - `.claude/` — Claude Code 技能配置

3. 【推荐】**不轻易新增顶层目录**：新增顶层目录意味着新增一个标签。只有当一批文档确实无法归入现有任何目录时，才创建新顶层目录。



### 目录层级约定

4. 【强制】顶层目录深度为 1 级，即 `标签/文档.md` 或 `标签/子目录/文档.md`。

   说明：不支持 `标签/子目录/子子目录/文档.md` 超过 2 层嵌套。如果需要更深层级，说明标签本身需要拆分。

5. 【推荐】子目录用于按主题细分，不作为独立标签。

   正例：`学习/Java学习/Spring.md` — "Java学习"是"学习"标签下的子主题，不独立成标签。
   
   反例：`Java学习/Spring.md` 单独作为顶层目录 — "Java学习"应该是"学习"的子集。



### 文档新增流程

6. 【强制】新增文档时必须确定归属标签（顶层目录），然后放入对应目录。

   步骤：
   
   1. 确定文档的主题
   2. 对照 `04-顶层目录标签定义.md` 找到匹配的顶层目录
   3. 如果匹配到多个目录，选最精确的那个
   4. 如果都不匹配且文档数量达到 3 篇以上，考虑新增顶层目录
   5. 按 `02-文档命名规范.md` 命名文件



### 文档迁移流程

7. 【推荐】从 `old/` 目录迁移文档到正式目录时，逐目录进行，每完成一个顶层目录提交一次。

   步骤：
   
   1. 明确当前要整理的顶层目录
   2. 从 `old/` 中找到该目录的所有文件
   3. 逐一确认文件是否属于当前标签
   4. 将确认的文件移到对应的顶层目录
   5. 提交 commit



### 标签冲突处理

8. 【推荐】当一个文档可能同时属于多个标签时，按以下优先级选择：

   1. 文档的**主要目的**是什么（学习？记录？作业？）
   2. 文档的**受众**是谁（自己复习？提交给老师？面试准备？）
   3. 文档的**时效性**（一次性作业 vs 长期维护的笔记）

   例：一篇关于 Spring 的作业报告 → 归入 `作业/`（主要目的是提交作业），而非 `学习/`（虽然也有学习成分）。



### 标签合并

9. 【推荐】当两个顶层目录的标签含义高度重叠时，应合并。合并判断标准：

   - 两个目录的文件互换位置后，目录名仍能准确描述内容
   - 两个目录的 inclusion criteria 重合度超过 50%
   - 至少一个目录文件数少于 5 篇

   当前建议合并的目录对：
   - `感想/` + `成长记录/` → `反思/`（均为个人反思类）
   - `AI资料/` + `AI踩坑/` → `卡片/`（均为原子知识）
   - `实习准备/` + `实习计划/` → `实习/`（边界模糊）
   - `项目/` + `我的项目/` → `项目/`（"我的"不增加区分度）
