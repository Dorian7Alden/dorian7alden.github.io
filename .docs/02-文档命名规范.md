## 文档命名规范



> 【强制】：必须遵守的内容
>
> 【推荐】：最好遵守的内容，特殊条件下可以自行调整



### 1. 通用规则

1. 【强制】文件名使用 `.md` 扩展名。

2. 【强制】禁止使用纯数字文件名。

   反例：`1.md`、`2.md`
   正例：`01-问题描述.md`、`2026-06-09-重庆本手.md`

3. 【强制】禁止文件名中出现以下 artifact：

   - 重复后缀（如 `-hard-hard`）
   - 多余的标点（如 `——` 结尾）
   - 拼写错误（如 `advance` 应为 `advanced`）

4. 【推荐】文件名中的中英文之间不加空格，中文与英文之间自然连接。

   正例：`01-AI-api解析失败.md`、`08-Spring.md`
   反例：`01-AI - api 解析失败.md`

5. 【推荐】文件名长度控制在 50 个字符以内，能准确概括文档主题。

6. 【推荐】`README.md` 仅用于项目子目录的入口说明，不作为普通文档名。



### 2. 卡片/ 目录

7. 【强制】命名格式：`NN-title.md`

   - `NN`：两位递增数字（01-99），新卡片取当前最大编号 +1
   - `title`：中文或中英混合的简短描述，概括卡片主题
   
   正例：`01-AI-api解析失败.md`、`31-一次课堂临时接手的 Streamlit 问卷平台网络卡顿排查记录.md`

8. 【强制】禁止使用日期作为卡片文件名。

   反例：`2026-04-27.md`



### 3. 学习/ 目录

9. 【推荐】命名格式：`{主题}.md`

   文件名使用中文描述性名称，准确反映文档涵盖的知识领域。

   正例：`Git.md`（根目录）、`Spring.md`（Java学习/）、`Java并发编程面试题.md`（小林coding/）
   反例：`main.md`、`经验.md`（过于宽泛）

10. 【推荐】子目录名使用中文，按学科或课程组织。

    正例：`Java学习/`、`小林coding/`、`计算机四大件/`



### 4. 非计算机/Daily English Dictation/

11. 【强制】命名格式：`NNN-level.md`

    - `NNN`：三位数字（001-999），表示第几期
    - `level`：难度等级，严格限定为以下四个值：
      - `beginner` — 入门
      - `intermedia` — 中级  
      - `advanced` — 高级
      - `expert` — 专家

    正例：`056-beginner.md`、`064-advanced.md`
    反例：`Dictation-024-hard-hard.md`（废弃的 Dictation- 前缀 + 重复后缀）、`054-advance.md`（拼写错误）

12. 【强制】废弃 `Dictation-NNN-suffix.md` 格式。所有文件统一为 `NNN-level.md`。



### 5. Competitions/ 目录

13. 【强制】禁止使用过于通用的文件名。

    反例：`题目.md`、`review.md`、`总论.md`
    正例：`图论与最短路径算法.md`、`25年MCM题目解析.md`

14. 【推荐】子目录按竞赛类型组织。

    正例：`Algorithm/`、`MCM/`、`UIEP/`



### 6. 作业/ 目录

15. 【推荐】命名格式：`{课程名}-{作业名}.md`

    正例：`web实验报告3.md`、`软工导论-第一学期作业汇总.md`
    反例：`c++实验.md`（缺少具体实验编号或主题）

16. 【强制】禁止文件名末尾出现多余标点。

    反例：`Harvard University Intro——.md`



### 7. 非计算机/日常学习卡片/

17. 【推荐】目录按 `{YY-MM}/week{N}/` 组织，文件名用 `day{N}.md`。

    正例：`26-03/week2/day1.md`



### 8. 招聘记录/ 目录

18. 【推荐】命名格式：`{YYYY-MM-DD}-{事件名}.md` 或 `{类型}-{描述}.md`

    正例：`2026-06-09-重庆本手.md`、`背景调研-重庆本手科技与仙桃数据谷.md`



### 9. 感想/ 与 成长记录/ 目录

19. 【推荐】使用编号前缀 + 描述性标题。

    正例：`01-AI 发展对大学生的影响.md`、`01-Git-pr-fix.md`



### 10. 项目/ 与 我的项目/ 目录

20. 【推荐】每个项目一个子目录，子目录中用描述性文件名或 `README.md`。

    正例：`项目/Harness/README.md`、`我的项目/Tetris.md`



### 附录：已识别的命名问题清单

整理文档时应修复以下问题：

| 文件 | 问题 | 修复为 |
|------|------|--------|
| `卡片/2026-04-27.md` | 日期命名 | `NN-title.md` |
| `学习/JsRE/main.md` | 过于通用 | `JsRE学习笔记.md` |
| `实习计划/问题整理/1.md`、`2.md` | 纯数字 | 加描述性标题 |
| `Competitions/MCM/*/题目.md` | 过于通用 | 加具体信息 |
| `Competitions/MCM/*/review.md` | 过于通用 | 加具体信息 |
| `作业/Harvard University Intro——.md` | 多余标点 | 去除 `——` |
| Dictation 中 `-advance` | 拼写错误 | `-advanced` |
| `Dictation-024-hard-hard.md` | 重复后缀 + 旧格式 | `024-hard.md` |
