## 顶层目录标签定义



本文档定义每个顶层目录作为"标签"的精确含义、归入条件和排除条件。



### AI/

- **标签含义**：人工智能相关所有内容。包括 AI 学习研究、提示词工程、AI 工具使用、AI 踩坑记录、AI 辅助开发。
- **归入条件**：
  - AI/ML 理论学习与架构分析
  - AI 框架使用（Spring AI 等）
  - 提示词工程与优化
  - AI 工具使用中的问题和解决方案
  - AI 辅助开发的经验和指南
  - AI 基础设施和工具配置
- **排除条件**：
  - 普通技术问题（不涉及 AI）→ 归入 `卡片/` 或 `学习/`
  - AI 话题的个人感想 → 归入 `AI/感想/`
- **子目录**：`学习/`、`踩坑/`、`提示词/`、`工具与基础设施/`、`辅助开发/`、`感想/`



### 卡片/

- **标签含义**：原子化知识卡片。记录一个独立的技术问题、解决方案或踩坑经验。
- **归入条件**：
  - 单个、自包含的问题-方案对
  - 简短的技术技巧或 CLI 使用提示
- **排除条件**：
  - 系统性的课程笔记 → 归入 `学习/`
  - 多段落个人反思文章 → 归入 `感想/`
  - 项目开发日志 → 归入 `项目/`
  - AI 相关问题 → 归入 `AI/`
- **子目录**：无（扁平编号列表）



### 学习/

- **标签含义**：系统性学习笔记。课程内容、技术深度研究、面试知识库、工具使用指南。
- **归入条件**：
  - 课程笔记（带有章节结构）
  - 技术参考手册（如 Git、Linux 用法）
  - 面试题整理（如小林coding 系列）
  - AI/ML 学习内容（已移至 `AI/` 顶层目录）
  - 计算机基础（四大件）
- **排除条件**：
  - 一次性作业提交 → 归入 `作业/`
  - 单次调试的简短记录 → 归入 `卡片/`
  - 为通过面试而做的特定练习 → 归入 `实习准备/`
  - AI 相关学习内容 → 归入 `AI/`
- **子目录**：`Java学习/`、`JsRE/`、`小林coding/`、`工具使用/`、`提示词/`、`计算机四大件/`



### JavaWeb学习/

- **标签含义**：Java Web 技术栈的专项学习路径。
- **归入条件**：
  - Tomcat、Servlet、Maven、MyBatis、SpringSecurity、Spring 的学习笔记
- **排除条件**：
  - 非 Web 相关的 Java 内容 → 归入 `学习/Java学习/`
- **子目录**：无（编号序列）



### 非计算机/

- **标签含义**：非计算机技术的内容。目前包含 Daily English Dictation 和日常学习卡片。
- **归入条件**：
  - 每日英语听写练习
  - 按日期组织的日常学习日志（跨学科的每日收获）
- **排除条件**：
  - 纯技术主题的独立笔记 → 归入 `学习/` 或 `卡片/`
- **子目录**：`Daily English Dictation/`、`日常学习卡片/`



### Competitions/

- **标签含义**：学术竞赛相关资料。算法竞赛、数学建模、UI/UX 项目比赛。
- **归入条件**：
  - 竞赛算法模板和题解
  - 数学建模方法总结和赛题分析
  - 竞赛项目文档
- **排除条件**：
  - LeetCode 面试练习 → 归入 `实习准备/`
  - 课程作业 → 归入 `作业/`
- **子目录**：`Algorithm/`、`MCM/`、`UIEP/`



### 作业/

- **标签含义**：大学课程作业。提交的实验报告、论文、作业汇总。
- **归入条件**：
  - 有明确课程归属的作业提交
  - 课程要求的实验报告
  - 课程论文和职业规划书
- **排除条件**：
  - 自发学习的课程笔记 → 归入 `学习/`
  - 个人反思（非作业要求） → 归入 `感想/`
- **子目录**：无（按课程平铺）



### 感想/

- **标签含义**：个人感想与观点文章。对技术、学习、生活的自由思考。
- **归入条件**：
  - 独立成篇的观点文章
  - 对技术趋势的个人看法
- **排除条件**：
  - 具体事件驱动的学习记录 → 归入 `成长记录/`
  - 课程要求的反思文章 → 归入 `作业/`
- **子目录**：无

> 【推荐】与 `成长记录/` 考虑合并为 `反思/`。



### 成长记录/

- **标签含义**：事件驱动的个人成长记录。记录了某个具体事件及从中获得的收获。
- **归入条件**：
  - 有明确触发事件（如一次 PR 修复、一次技术分享）
  - 包含"背景 → 问题 → 解答 → 收获"的叙事结构
- **排除条件**：
  - 抽象观点文章 → 归入 `感想/`
  - 纯技术方案记录 → 归入 `卡片/`
- **子目录**：无

> 【推荐】与 `感想/` 考虑合并为 `反思/`。



### 项目/

- **标签含义**：项目相关的文档。包含个人项目的开发记录、工具配置、项目灵感。
- **归入条件**：
  - 个人项目的开发文档和架构设计
  - AI 工具和技能的配置说明（Harness、Skill）
  - 项目灵感记录
- **排除条件**：
  - 竞赛项目 → 归入 `Competitions/`
  - 课程项目 → 归入 `作业/`
  - AI 工具与技能配置 → 归入 `AI/工具与基础设施/`
- **子目录**：按项目分（`Harness/`、`Skill/`、`TODO可视化时间表/`、`英语学习app/`）



### 我的项目/

- **标签含义**：已实际构建的个人项目。区别于 `项目/` 的"想法和工具"。
- **归入条件**：
  - 有实际代码、已部署或完整开发的项目
- **排除条件**：
  - 项目灵感和未启动的想法 → 归入 `项目/`
- **子目录**：按项目分（`Tetris/`、`樟木林/`）

> 【推荐】与 `项目/` 考虑合并。当前区分度不够，'我的'不增加信息量。



### 招聘记录/

- **标签含义**：招聘会参与记录。公司调研、面试经历、学生感言。
- **归入条件**：
  - 招聘会现场记录
  - 公司背景调研
  - 面试经历和感言
- **排除条件**：
  - 面试准备和刷题 → 归入 `实习准备/`
  - 实习规划 → 归入 `实习计划/`
- **子目录**：无



### 实习准备/

- **标签含义**：实习求职的准备工作。算法刷题、面试题、开发规范。
- **归入条件**：
  - LeetCode 题解
  - 面试题收集和整理
  - 开发规范文档
- **排除条件**：
  - 实际面试记录 → 归入 `招聘记录/`
  - 实习规划和策略 → 归入 `实习计划/`
- **子目录**：`LeetCodeHot100/`、`开发规范/`、`面试题/`



### 实习计划/

- **标签含义**：实习过程的规划文档。学习计划、能力要求、问题整理。
- **归入条件**：
  - 实习前的学习规划
  - 实习中遇到的问题整理
  - 实习能力目标和策略
- **排除条件**：
  - 面试准备内容 → 归入 `实习准备/`
  - 实习面试记录 → 归入 `招聘记录/`
- **子目录**：`算法题/`、`问题整理/`

> 【推荐】与 `实习准备/` 考虑合并为 `实习/`。
